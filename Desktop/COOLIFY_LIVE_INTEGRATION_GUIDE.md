# 🚀 Coolify MCP Live Integration Guide

## 🎯 Overview

This guide explains how to take your Coolify MCP integration live by connecting the MCP server to your actual Coolify instance.

## 🔧 Current Implementation Status

The MCP system currently has two modes:

1. **Simulation Mode (Current Default)** - Simulates Coolify API responses for safe testing
2. **Live Mode** - Connects to real Coolify API when properly configured

## 🛠️ Going Live: Step-by-Step

### 1. Configure Environment Variables

Set these environment variables to enable live Coolify integration:

```bash
# Set Coolify API endpoint
export COOLIFY_API_URL="https://your-coolify-instance.com"

# Set your Coolify API key
export COOLIFY_API_KEY="your-coolify-api-key-here"

# Enable real API mode
export ENABLE_REAL_API="true"
```

**Windows (Command Prompt):**
```cmd
set COOLIFY_API_URL=https://your-coolify-instance.com
set COOLIFY_API_KEY=your-coolify-api-key-here
set ENABLE_REAL_API=true
```

**Windows (PowerShell):**
```powershell
$env:COOLIFY_API_URL="https://your-coolify-instance.com"
$env:COOLIFY_API_KEY="your-coolify-api-key-here"
$env:ENABLE_REAL_API="true"
```

### 2. Obtain Coolify API Credentials

1. Log in to your Coolify dashboard
2. Navigate to **Settings** > **API Keys**
3. Generate a new API key with appropriate permissions
4. Copy the API key for use in the environment variable

### 3. Start the MCP Server

```bash
python coolify-mcp-server.py
```

You should see:
```
Real API Integration: ENABLED
```

### 4. Configure MCP Client

Update your `coolify-mcp-client.py` with the API key generated by the server:

```python
# In coolify-mcp-client.py
SECRET_KEY = "your-server-generated-api-key-here"  # Must match server's secret key
```

### 5. Test the Integration

```bash
# Create a test resource
python create_trading_bot.py

# Check server logs for real API responses
```

## 🔄 Switching Between Modes

### Disable Live Mode (Return to Simulation)

```bash
export ENABLE_REAL_API="false"
# or unset ENABLE_REAL_API
```

### Re-enable Live Mode

```bash
export ENABLE_REAL_API="true"
```

## 📋 API Endpoints Configuration

The MCP server maps MCP commands to Coolify API endpoints:

| MCP Command | Coolify API Endpoint | HTTP Method |
|-------------|----------------------|-------------|
| CREATE      | `/api/resources`      | POST        |
| UPDATE      | `/api/resources/{id}`| PUT         |
| DELETE      | `/api/resources/{id}`| DELETE     |
| LIST        | `/api/resources`      | GET         |

## 🔒 Security Considerations

### 1. API Key Security
- Never commit API keys to version control
- Use environment variables or secret management systems
- Rotate API keys regularly

### 2. Network Security
- Ensure your Coolify instance uses HTTPS
- Configure proper firewall rules
- Consider using VPN or private networking

### 3. MCP Server Security
- Keep `SECRET_KEY` secure and consistent between client and server
- Run MCP server on a secure port
- Consider adding TLS/SSL encryption

## 🚨 Troubleshooting

### Common Issues & Solutions

**Issue: "Real API enabled but no Coolify API key configured"**
- Solution: Set the `COOLIFY_API_KEY` environment variable

**Issue: Connection refused or timeout errors**
- Solution: Verify Coolify API URL is correct and accessible
- Check network connectivity and firewall settings

**Issue: Authentication failed**
- Solution: Verify your Coolify API key is valid
- Check for typos in the API key

**Issue: Resource creation fails**
- Solution: Check Coolify API response in server logs
- Verify your Coolify instance has available resources
